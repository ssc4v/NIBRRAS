Enhance the Learning module by adding a Knowledge Relationship Map.

Important:
- Keep mock data only.
- No real API calls.
- No external services.
- No paid services.
- Arabic RTL.
- Black-and-white monochrome theme.
- Mobile-first.
- Strict Front-end / Back-end separation.
- UI must call learningService only.

Goal:
NIBRRAS must have two connected learning views:

1. خريطة التعلم
A hierarchical learning tree:
Domain → Area → Branch → Sub-Branch → Lesson → Questions → Reviews

2. خريطة العلاقات
A graph/network map showing relationships between every saved knowledge item.

Difference:
خريطة التعلم = ترتيب هرمي للتعلم.
خريطة العلاقات = شبكة تربط المعلومات ببعض حتى لو كانت في مجالات مختلفة.

Add inside Learning tab a new internal section:
"خريطة العلاقات"

Now Learning tab internal sections should be:
1. خريطة التعلم
2. خريطة العلاقات
3. بنك الأسئلة
4. مولّد الأسئلة
5. ملفات التعلم

- target node
- relationship type
- strength: ضعيف / متوسط / قوي
- confidence score

Design rules:

- Secondary buttons must be black with gray border.

5. Questions are linked to concepts.
6. Files are linked to concepts.
